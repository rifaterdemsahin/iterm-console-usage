# Claude Code Instructions

## Output Workflow

After creating any output document (e.g. research summaries, feature overviews, comparison docs), always:

1. `git add` the new file
2. `git commit` with a descriptive message
3. `git push` to the remote

This applies to any `.md` or other output file written as the result of a user request.

## Commit Style

- Short, imperative subject line (e.g. `Add Warp AI features overview`)
- No trailing summaries needed in the commit body
